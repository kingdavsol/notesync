Security at NoteSync

We take the security of your notes and personal data seriously.

Encrypted in Transit

All data between the app and our servers is encrypted using TLS 1.2+ (HTTPS). Your notes never travel over unencrypted connections.

Password Hashing

Passwords are hashed using bcrypt (cost factor 12). We never store or log your plaintext password â€” not even we can read it.

JWT Authentication

Sessions use short-lived JSON Web Tokens signed with a server-side secret. Tokens expire automatically to minimize exposure if intercepted.

Rate Limiting

Login and registration endpoints are rate-limited to 10 requests per 15 minutes per IP to prevent brute-force and credential-stuffing attacks.

Input Sanitization

All user input is sanitized server-side to prevent XSS and injection attacks. SQL queries use parameterized statements (no string interpolation).

Security Headers

Our API uses Helmet.js to set strict HTTP security headers including Content-Security-Policy, X-Frame-Options, and X-Content-Type-Options.

Infrastructure Security

  • Servers run on dedicated VPS infrastructure (Contabo, Germany) â€” not shared hosting.
  • SSH access is key-based only; password authentication is disabled.
  • The PostgreSQL database is not exposed to the public internet â€” only accessible from localhost.
  • Nginx reverse proxy sits in front of the Node.js application server.
  • SSL/TLS certificates are managed by Let's Encrypt with automatic renewal.
  • The application runs as a non-root user via PM2 process management.

Data Protection

  • User passwords: bcrypt with salt (cost factor 12).
  • Database credentials: stored in environment variables, never in code.
  • JWT secret: stored in environment variables, rotated periodically.
  • Audio for transcription: transmitted to Groq API over HTTPS; audio files are never stored on our servers.
  • Deleted notes: soft-deleted immediately; purged from backups within 90 days.

What We Don't Do

  • We do not store plaintext passwords.
  • We do not sell or share your note content with third parties.
  • We do not use advertising SDKs or tracking pixels.
  • We do not retain voice recordings after transcription.
  • We do not access your notes for any purpose other than serving them back to you.
